# 함수의 본질 🧠
재귀함수를 이야기 하기전, 함수의 본질에 대해 자세히 알 필요가 있다.  
**함수는 왜 사용하는가?** 아마 여러분의 대부분은 함수를 **'명령을 묶는 장치'** 정도로 생각하고 문제를 풀때 활용했을 것이다.  
맞는 말이다. 정확한 정의이기도 하다. 하지만 이렇게만 이해한다면 절대 재귀함수를 이해할 수 없다.  
이렇게 함수를 이해했으니 '재귀함수는 사실 점화식이다' 라고 이야기 해줘도 이해 못하는 것이다.

![](./images/function.png)
따라서 다시 함수를 재정의를 해주자.  
함수란 **특정 작업을 수행하게 하거나 특정 값을 반환받아 사용하기 위해 사용하는 코드의 묶음**이다.  
즉, 모든 함수는 **"값을 반환하는 함수"** 와 **"행동을 수행하는 함수"** 로 분류할 수 있다.  

## a. 값을 반환하는 함수
값을 반환하는 함수는 말 그대로 함수 내에서 **계산을 수행, 그 계산 값을 반환**해준다. 예시를 살펴보자.
```c
int sum(int a, int b) {
    return a+b;
}
```
이 함수는 인자로 `int` 형식의 변수 `a`와 `b`를 입력받았고 `a+b`를 반환했다.  
여기에서 집중해야할 포인트는 함수를 선언할 때 반환하는 값의 형식과 함수의 형식이 일치한다는 것과 `return` 뒤에 오는 값이 있다는 것이다.

## b. 행동을 수행하는 함수
행동을 수행하는 함수는 **아무런 값을 반환하지 않으며, 행동만을 수행**한다. 예시를 살펴보자.
```c
void sum(int a, int b) {
    printf("%d", a+b);
    return;
}
```
이 함수는 인자로 `int` 형식의 변수 `a`와 `b`를 입력받았고 `a+b`를 출력했다.  
여기에서 집중해야할 포인트는 함수를 선언할때 `void`를 사용하였으며 `return` 뒤에 오는 값이 없다는 것이다.

# 재귀함수랑 인사하기 👋🏻
![](https://pbs.twimg.com/media/Ee9692iUYAENVo0.jpg)
재귀함수란 **자기 자신을 호출하는 형태의 함수**이다. 재귀함수 또한 함수의 일종이므로 앞서 배웠던 것이 적용된다.  
다시 한번 복습해보자.
>1. **값을 반환하는 함수** : 반환하는 값이 있다. 함수의 형식이 있다. `return` 뒤에 값이 온다.
>2. **행동을 수행하는 함수** : 반환하는 값이 없다. 함수의 형식이 없다. `return` 뒤에 값이 오지 않는다.

재귀함수에서 가장 중요한것은 **종료조건과 재귀문**이다. 이에 대해 알아보기 위해 다음 코드를 보자.
```c
int sum(int n) {
    if(n==1)
        return 1;
    return n+sum(n-1);
}
```
이 코드에서 종료조건은 `if(n==1)`에 해당하며 재귀문은 `return n+sum(n-1)`에 해당한다.  
이 두가지에 집중하여야 재귀함수를 쉽게 만들 수 있다.  

## a. return의 재정의
앞서 함수를 재정의했듯 말그대로의 '반환하다'의 의미로 이해했던 return 문을 다시 재정의 해보자.  
return은 **값을 반환하거나 바프로그램을 종료시키는 구문** 이다.
>1. **값을 반환하는 return** : 결과값을 반환한다. 값을 반환하는 함수에서만 사용된다.
>2. **종료의 의미로 사용되는 return** : 결과값을 반환하지 않는다. 함수의 실행을 종료시킬때 사용된다.

## b. 값을 반환하는 재귀함수
**값을 반환하는 재귀함수는 함수 내에서 함수값을 활용한다.**  
우리가 문제에서 만나는 대부분의 함수는 값을 반환하는 재귀함수다.  

위에서 봤던 예시를 다시 보자.
```c
int sum(int n) {
    if(n==1)
        return 1;
    return n+sum(n-1);
}
```
이 함수는 1부터 n까지의 합을 구하는 함수이다.  
이 함수에서는 `return`에서 `sum(n-1)`의 **함수값을 활용**하고 있음을 볼 수 있다.

![](./images/function2.png)
즉, 위 그림과 같은 과정을 통해 n까지의 함이 구해진다.  
이 과정은 이런 **점화식**으로 간단하게 나타낼 수 있다.  
><Latex>$S_1=1$</Latex>  
><Latex>{"$S_n=n+S_{n-1}$ $(n>1)$"}</Latex>  

이렇게 **점화식으로 표현하면 종료조건과 재귀문을 놓치지 않고 재귀함수로 표현하기 편리하다.**

[(재귀함수의 점화식 표현에 대한 자세한 내용은 여기서 확인하자)](/jaegui)


# c. 행동을 수행하는 재귀함수
**행동을 수행하는 재귀함수는 return을 종료조건으로 활용한다.**

다음 예시를 보자.  
```c
void prt(int n) {
    printf("%d", n);
    if(n==1)
        return;
    prt(n-1);
}
```
위 함수는 n부터 1까지의 수를 출력하는 재귀함수이다.  
이 함수에서는 `return`이 함수의 종료문 `if(n==1)`에서 **'종료의 의미 return'**으로 활용되고 있음을 알 수 있다.

행동을 수행하는 재귀함수는 함수값이 없기 때문에 점화식으로 나타낼 수 없다.  
따라서 행동을 수행하는 재귀함수는 직접 머리로 수행해보는 과정이 가장 중요하다.

# Wrap up 📖
지금까지 배운 내용들을 복습해보자.

### a. 함수의 분류
>1. **값을 반환하는 함수** : 반환하는 값이 있다. 함수의 형식이 있다. `return` 뒤에 값이 온다.
>2. **행동을 수행하는 함수** : 반환하는 값이 없다. 함수의 형식이 없다. `return` 뒤에 값이 오지 않는다.

### b. return의 기능
>1. **값을 반환하는 return** : 결과값을 반환한다. 값을 반환하는 함수에서만 사용된다.
>2. **종료의 의미로 사용되는 return** : 결과값을 반환하지 않는다. 함수의 실행을 종료시킬때 사용된다.

### c. 재귀함수를 이용한 문제 해결
>1. **값을 반환하는 재귀함수** : 점화식으로 표현하여 놓치지 말고 푼다.
>2. **행동을 수행하는 재귀함수** : 직접 머리로 수행해본다.
